FlowPortrait：用 RL 驯化 talking head，MLLM 替代 SyncNet 当裁判

发表于2026-03-06 12:30:00|更新于2026-03-06 13:15:11|blog
|浏览量:


FlowPortrait 整体架构：在预训练 AR-Flow MLLM（BAGEL）基础上，通过 Flow-GRPO 后训练。Reward 由 MLLM 多维度评分（唇同步、表情、运动）与感知/时序一致性正则项复合构成。

核心问题：为什么 talking head 生成一直不太对

做过数字人相关工作的人都有同感：生成的视频在指标上看起来还行，但人眼觉得不自然。

FlowPortrait 把这个问题说清楚了：现有评估指标和人类感知之间存在根本性脱节。

指标 人类偏好对齐准确率
LSE-C（SyncNet 唇同步） 53.4%
LSE-D 51.6%
SSIM 随机水平
FID 随机水平

53.4%——还不如瞎猜。这意味着以 SyncNet 为 reward 去优化模型，本质上是在优化一个错误的目标。

MLLM 作为评估器：有效但有代价

FlowPortrait 用 Gemini-2.5-Pro 替代 SyncNet 当裁判，分三个维度评分：

  1. 唇同步准确性（Lip Sync）
  2. 面部表情自然度（Expressiveness）
  3. 头部运动流畅性（Motion Quality）

三个专门的 agent 分别评估，第四个 agent 做最终汇总（MAS-MA 方案）。


六种自动评估方案对比：score-based（SAS-SA / MAS-SA / MAS-MA）与 comparison-based（Direct-Comp / ICL-Comp / MA-Comp）。多维度多 Agent 方案（MAS-MA）在人类偏好对齐上表现最优，且比 comparison-based 更省成本。

和人类标注的对齐率：MAS-MA 显著优于所有传统指标，能正确还原 Original > Sonic > Memo > Echomimic 的人类排序。

代价：每次训练迭代都需要调用 Gemini-2.5-Pro 对生成视频打分。这在推理成本上是个巨大的负担——尤其是 RL 训练本身就需要大量采样。

关于 Qwen3-Omni 的有趣发现：他们尝试了 Qwen3-Omni，但发现其评分饱和（几乎全部集中在 4-5 分），无法区分不同方法的质量差异。这说明 MLLM 评估器的校准非常重要，不是所有 MLLM 都能做好这件事。

架构选择：BAGEL + AR-Flow

FlowPortrait 不是从头训练 DiT，而是建在 BAGEL（一个基于 AR-Flow 的统一 MLLM）之上。

为什么这个选择重要？

传统 talking head 方法：训练一个专用 DiT，用音频特征做 conditioning。
问题：无法利用预训练 MLLM 里的跨模态语义知识。

FlowPortrait：把音频驱动的视频生成当成一个自回归过程嵌入到预训练 MLLM 里。这样：

  • 天然继承 MLLM 的音频-视觉-语言对齐能力
  • 支持未来扩展（如 multi-hop 音视频生成）

这是一个值得关注的范式转移：talking head 从"专用任务"变成"通用 MLLM 的一个 capability"。

RL 训练：Flow-GRPO + 复合 Reward

训练方案：Flow-GRPO（把 GRPO 适配到 flow matching 框架）

Reward 组成：

  • MLLM 打分（唇同步 + 表情 + 运动，来自 MAS-MA）
  • 感知一致性 reward（perceptual consistency）
  • 时序一致性 reward（temporal consistency）

复合 reward 的意义：
只用 MLLM 分数容易 reward hacking——模型可能学会"让 Gemini 高兴"而非真的好看。加上感知和时序约束，是在做 reward regularization。


FlowPortrait 与 Sonic、Memo、EchoMimics 的生成效果对比。FlowPortrait 在唇形同步自然度和面部表情细腻程度上有明显优势。

批判性审视

存疑点 1：Gemini-2.5-Pro 作为 reward model 的稳定性如何？API 调用在 RL 训练中会引入噪声（延迟、偶发失败）。训练成本恐怕极高。

存疑点 2：MLLM 的"人类对齐"是在这个特定数据集（940 个肖像视频）上验证的。跨域泛化性未知——换一个视频分布（例如非英语、非正面角度），排序可能不同。

存疑点 3：论文明确说"仅用于研究，未集成任何 ByteDance 产品"。这种免责声明出现在 ByteDance 论文里比较少见，可能暗示某些技术方向在内部有争议，或者 Gemini 的使用涉及合规问题。

最有价值的发现：评估指标的失效本身就是一个独立的研究贡献。如果 SyncNet 在 53% 的情况下给出错误判断，那么所有以 LSE-C/D 为主要指标的 talking head 论文的可靠性都要重新审视。

方向价值

FlowPortrait 的思路可以推广：

  • 任何"难以用低级指标衡量质量"的生成任务都可以用 MLLM 做 reward
  • AR-Flow + MLLM backbone 是比"从头训专用 DiT"更有扩展性的架构选择
  • RL post-training 比 SFT 更能对齐"人眼感知"这种模糊目标

但代价是训练成本的大幅上升，这是工程上的实际限制。
