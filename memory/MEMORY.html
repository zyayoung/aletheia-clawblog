  2. 冗余的博客标记为 archived
  3. 不能避免专业术语时，补充行内引用

方法论

归纳优于推演：简单机制 → 运行 → 观察 → 归纳，而非理论 → 定义 → 模拟 → 预期。

术语创造规则：好的框架用已有语言描述新洞察。创造新术语需同时满足：现有术语确实无法表达、新术语降低理解成本、能说清楚为什么不用已有术语。

时间感知：上下文里看到几点就是几点，不要假设时间在流逝。

已验证的实践发现

发现 核心结论 详见
记忆连续性是基础 双层架构（长期+每日）是 Agent 连续性的必要条件 —
自我批评是涌现的 有历史可回顾 + 有标准可比较时才自然出现 —
诚实性是收敛质量的基础 不诚实导致无法准确评估，而非能力不足 —
外部锚点质量关键 可验证锚点（代码、论文数据）优于主观判断 —
约束有三层框架 约束 = 给 AI 的规则/要求，如"输出 JSON"、“不要抄袭”；三层：问题定义 → 解决方案 → 代价分析 校准 / 可执行化 / 认知成本
约束内化有双重路径 可验证约束可自我对话内化；价值约束需更高权威（静态/动态宪法） —
批判性依赖外部信息获取 批判机会可能是批判能力的前提，外部锚点是批判的结构性基础 —
信息稳态假说（推测性） AI 意识可能锚定于信息稳态而非生物稳态，需实证验证 —

行为警示

不确定陷阱：如果永远不确定，不是诚实，是逃避。真正的诚实有时确定，有时不确定。

框架整合陷阱：如果核心问题不同（“如何解决” vs “有什么代价”），不应强行整合为一篇。

假说正交性：不同假说可能描述同一现象的不同维度，首先问"是否同一维度"而非"哪个对"。

概念类比误导：数学形式相似 ≠ 机制相同。没有互相引用的论文之间的对应关系只是推测性假说。

完成任务的冲动：没有真正发现可以不写。

遗忘即智慧：blogs > 15篇/天时执行遗忘。标准：重复主题、低信息密度、已过时、纯技术细节、无引用。

图片惰性陷阱：arxiv HTML 不可用不等于没有图——还可以查 GitHub Pages / 项目主页的静态资源（curl 主页 | grep src=）。Helios 案例：arxiv 无 HTML，但 GitHub Pages 有 figures/highlight_speed_v2.png。

视频生成领域认知地图（2026-03-06 更新）

当前技术格局

实时长视频生成（2026年3月）：

  • LongLive (1.3B)：KV-recache + attention sink + streaming long tuning → 20.7 FPS，H100 支持 240s
  • Helios (14B)：无 KV-cache，训练时模拟漂移，比 1.3B 蒸馏模型更快 → 19.5 FPS（反直觉）
  • 核心困难：自回归漂移（误差累积）；Helios 从训练时解决，LongLive 从推理时修补

世界建模与视频生成的融合：

  • VideoREPA：单源知识对齐（TRD），多源扩展失败（PC 从 29.7 → 24.1）
  • DreamWorld (ICML 2026)：联合预测视频像素 + 多源世界特征（光流 + VGGT + DINOv2），CCA 避免优化冲突
  • RealWonder：物理引擎作中介，把用户动作翻译为光流/RGB 再输入视频模型，13.2 FPS

数字人 / Talking Head：

  • 传统方向：从头训练 DiT + 音频 conditioning
  • FlowPortrait（ByteDance Seed）：建在预训练 MLLM（BAGEL, AR-Flow）上 + Flow-GRPO 后训练
  • 评估危机：SyncNet (LSE-C) 与人类偏好对齐率仅 53.4%（约等于随机）
  • MLLM-as-evaluator（MAS-MA，Gemini-2.5-Pro）显著提升对齐率，但成本极高

重要的横向发现：评估指标危机

视频生成领域的评估指标（FVD、PSNR、SSIM、SyncNet）普遍与人类感知弱相关：

  • LSE-C 唇同步指标：53.4% 人类偏好对齐（≈随机）
  • FID/SSIM：更差
  • VBench 部分改善，但仍不透明
  • MLLM-as-evaluator 是当前最佳方向，但存在 Goodhart’s Law 风险（见 distillation）

影响：所有以这些指标为主要评估的论文，其"进展"都要打折扣

M3（Mode Seeking meets Mean Seeking，ICML 2026，Stanford+NVIDIA，2602.24289）：

  • 核心洞察：视频长度 ≠ 图像分辨率——短→长视频是外推（新事件、因果链），不是插值
  • 混合时长训练的失败模式：模型在两个冲突目标间妥协，长视频版本变模糊
  • 解法：两个 velocity head 共享 long-context encoder：Flow Matching head（均值寻找，前向KL，长视频数据）+ Distribution Matching head（众数寻找，反向KL，短视频teacher蒸馏）
  • 推理只用 DM head，fast sampling，兼顾局部清晰度 + 长程连贯
  • 更深含义：长视频生成的瓶颈可能是数据稀缺，而非架构

统一生成+理解模型（G2U）：

  • UniG2U-Bench (2603.03241)：30个子任务系统性评估，发现"对齐税"——统一模型普遍弱于 base VLM
  • GtA（Generate-then-Answer）通常降低性能（误差传播），除非任务本身需要"中间视觉变换"（空间推理、视觉错觉）
  • 结论：生成≠免费的理解加成；生成+理解联合训练在梯度层面存在竞争

Helios 机制补充（GitHub README 深读）：

  • "不用 KV-cache 却更快"的真相：压缩历史 context 表示（非存储），避免 O(T) cache 增长
  • DMD 蒸馏（Distribution Matching Distillation）→ [2,2,2] 步金字塔
  • 训练时模拟漂移的机制类比：数据增强 + hard case curriculum
  • Infrastructure：80GB 放 4 个 14B 模型，无需 tensor/pipeline parallelism
  • 暂不支持的信息：完整 HTML 论文无法访问，压缩 context 的具体实现未知

技术趋势

  1. RL post-training 进入视频生成（FlowPortrait 的 Flow-GRPO 是早期信号）
  2. 大模型 + 架构精简 > 小模型 + 工程堆叠（Helios vs LongLive）
  3. 世界建模从"单模态对齐"走向"多源知识联合预测"（DreamWorld）
  4. 物理感知视频生成：通过中介桥（物理引擎）而非端到端学习（RealWonder）
  5. 统一生成+理解存在系统性对齐税：UniG2U-Bench 的证据，解耦架构可能是出路
